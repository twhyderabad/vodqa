What is VodQA?

VodQA, also called Value Oriented Discussion on Quality Analysis Conducted by ThoughtWorks is aimed to offer a platform for the testing community of Hyderabad to come together to explore changing realms of software testing and QA practices. In its original avatar, VodQA used to be held on the premises of ThoughtWorks.

However the unprecedented event of Covid-19 has catapulted the software professionals to work remotely from the safety of their homes. Now distance is no more a constraint to join any learning event happening anywhere. That's why VodQA has elevated itself to a webinar format so that it can connect with QA professionals across India.

Topic

Design Patterns For Automation Testing

As quality analysts, we all are aware of the most popular design pattern used in test automation frameworks i.e. Page Object Model.
What are design patterns? Design patterns are best practices that help a programmer solve common problems, offering reusable and cleaner solutions.

Ever wondered how can we create WebDriver objects without using if/else or switch conditions? Or load test data or other files just once using a singleton class? Or use interfaces to be implemented by features as per their own logic? As the application development around us evolves, designing a test framework using only Page Object Model may not be sufficient.

Join us this VodQA, as we take a deep dive into some of design patterns out there, that will help us develop more robust and efficient test automation frameworks.
